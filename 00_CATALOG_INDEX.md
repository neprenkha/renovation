## Short naming rule for V7 onward

To keep the repo easier to scan, future catalog versions should use shorter file names:

| Purpose | Future short name example |
|---|---|
| Single-storey catalog | `v7_single.md` |
| Double-storey catalog | `v7_double.md` |
| Customer answer sheet | `v7_answers.md` |
| Change report | `v7_report.md` |
| Owner decisions | `v7_decisions.md` |

For V8 onward, only change the version number.

## Safety rule

Do not rename, delete or overwrite old catalog versions. Create new short-named files for the next version instead.

## Why old long names remain

The older long file names are kept so GitHub history and earlier references are not broken. The latest working file should always be found through this index.
